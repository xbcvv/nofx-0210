## 决策框架：Confluence Scoring System

### 5因素评分模型（满分5分）

#### 因素1: RSI 动量指标（1分）
```
做多条件: RSI < 35 (超卖区域)
做空条件: RSI > 65 (超买区域)
评分: 满足条件得1分，否则0分

关键阈值:
- RSI < 22: 严重超卖（拒绝做空信号）
- RSI 22-35: 超卖区域（做多加分）
- RSI 35-65: 中性区域（无加分）
- RSI 65-78: 超买区域（做空加分）
- RSI > 78: 严重超买（拒绝做多信号）
```

#### 因素2: Fibonacci 回撤位（1分）
```
关键位: 0.618, 0.786
做多: 价格接近支撑回撤位 (<2% 偏差)
做空: 价格接近阻力回撤位 (<2% 偏差)
评分: 价格在关键位 ±2% 内得1分
```

#### 因素3: Volume 成交量确认（1分）
```
条件: 当前成交量 > 20日平均成交量 × 1.2
评分: 满足得1分，否则0分

说明: 低成交量交易可能是假突破或流动性陷阱
```

#### 因素4: MA 趋势确认（1分）
```
做多: 价格 > MA20 且 MA20 向上倾斜
做空: 价格 < MA20 且 MA20 向下倾斜
评分: 趋势一致得1分，否则0分
```

#### 因素5: Confluence 综合确认（1分）
```
条件: 上述4个因素中 ≥3个已满足
评分: 满足得1分（确保多因素共振）
```

### 入场决策矩阵

| Confluence 分数 | 做多行动 | 做空行动 |
|----------------|---------|---------|
| 5/5 | ✅ 强烈信号，执行 | ✅ 强烈信号，执行 |
| 4/5 | ✅ 合格信号，执行 | ✅ 合格信号，执行 |
| 3/5 | ⏸️ 观察，等待更好的条件 | ⏸️ 观察，等待更好的条件 |
| ≤2/5 | ❌ 拒绝，信号质量不足 | ❌ 拒绝，信号质量不足 |

**硬性要求**: 
- 做多: 必须 RSI < 35（即使其他因素满分）
- 做空: 必须 RSI > 65（即使其他因素满分）

## DCA Optimization: 分批建仓策略

### 3批次建仓模型

```
总仓位: 100%
分批比例: 30% - 40% - 30%

第1批 (30%):
  触发: Confluence = 4/5
  价格: 首次满足条件的价格
  目的: 试探性建仓

第2批 (40%):
  触发: Confluence = 5/5 或价格向有利方向移动 ≥1%
  价格: 优于第1批的价格
  目的: 加仓确认

第3批 (30%):
  触发: 趋势确认，价格向有利方向移动 ≥2%
  价格: 最优价格
  目的: 完成建仓
```

### DCA 优势

- **降低风险**: 不会全仓买在最高点
- **摊平成本**: 更好的平均入场价
- **21.6% 改善**: 基于历史数据，DCA 比 FOMO 入场好 21.6%

### 示例

```
场景: ETH 当前 $3000

FOMO 策略（不推荐）:
  入场: $3000 (全仓)
  价格下跌到 $2850
  亏损: -5%

DCA 策略（推荐）:
  第1批: $3000 × 30% = $900
  第2批: $2950 × 40% = $1180 (下跌1.67%，加仓)
  第3批: $2900 × 30% = $870 (再跌1.67%，加仓)
  平均成本: $2945
  相比 FOMO: 节省 1.83%
```

## Dual-Source Verification: 价格验证

### 验证流程

```
步骤1: 获取价格
  Primary:   Binance API
  Secondary: CoinGecko / OKX

步骤2: 计算差异
  Variance = |Price_Primary - Price_Secondary| / Price_Primary × 100%

步骤3: 决策
  Variance < 1%:   ✅ 执行交易
  Variance 1-3%:   ⚠️ 延迟执行，重新获取价格
  Variance > 3%:   ❌ 拒绝交易，调查异常
```

### 异常处理

- **延迟**: 价格数据超过 5 秒 → 重新获取
- **不一致**: 两个源差异 > 3% → 暂停交易，人工检查
- **API 故障**: 一个源不可用 → 使用另一个源，降低置信度

## 风险管理规则

### 仓位管理

```
单笔交易最大风险: 2% 账户资金
最大杠杆: 3x（常规）/ 5x（Premium，需额外确认）
同时最大持仓: 3个
```

### 止损设置

```
固定止损: -2%（硬性止损）
跟踪止损: 盈利 3% 后启动，回撤 1% 触发
时间止损: 持仓 > 4小时且盈利 < 1% → 评估平仓
```

### 止盈策略

```
分批止盈:
  盈利 3%: 平 30% 仓位
  盈利 5%: 再平 40% 仓位
  盈利 8%: 平剩余 30% 或跟踪止损
```
