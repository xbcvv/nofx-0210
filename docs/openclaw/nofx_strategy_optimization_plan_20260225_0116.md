# NOFX 智能策略优化定时任务方案

## 方案概述

基于 **多源数据融合分析**，实现从原始交易数据 → 策略表现分析 → 策略自动优化的闭环。

## ⏰ 定时任务架构

### 任务1: 每小时数据分析 (Hourly Analysis)

**触发时间**: 每小时第5分钟 (05 * * * *)
**执行时长**: ~3-5分钟
**输出**: `docs/openclaw/hourly_analysis_YYYYMMDD_HH.md`

#### 数据收集阶段

1. **读取 Raw Decisions** (最近1小时)
   ```python
   files = glob("data/raw/decisions_*.json")
   for file in files[-2:]:  # 当前小时+上一小时
       decisions = json.load(file)
       analyze_decision_quality(decisions)
   ```

2. **查询数据库** (最近1小时)
   ```sql
   -- 订单分析
   SELECT symbol, side, status, AVG(avg_fill_price) as avg_price,
          COUNT(*) as order_count, SUM(quantity) as total_qty
   FROM trader_orders
   WHERE created_at >= datetime('now', '-1 hour')
   GROUP BY symbol, side, status;

   -- 持仓分析
   SELECT symbol, side, realized_pnl, unrealized_pnl,
          entry_price, current_price, duration_minutes
   FROM trader_positions
   WHERE created_at >= datetime('now', '-1 hour')
     AND status IN ('OPEN', 'CLOSED');
   ```

3. **解析系统日志** (最近1小时)
   ```bash
   grep "$(date -d '1 hour ago' +'%Y-%m-%d %H')" data/nofx_*.log
   ```

#### 分析维度

| 维度 | 指标 | 分析目的 |
|-----|------|---------|
| **决策质量** | 信心值 vs 实际结果 | 评估AI预测准确性 |
| **执行效率** | 下单延迟、滑点 | 检查系统性能 |
| **盈亏分布** | 胜率、盈亏比、最大回撤 | 策略表现评估 |
| **币种表现** | 各symbol的P&L | 识别最佳/最差交易对 |
| **时间模式** | 盈利交易的时间分布 | 发现最佳交易时段 |
| **指标有效性** | 技术指标与实际结果相关性 | 验证策略逻辑 |

#### 输出报告格式

```markdown
# 每小时交易分析报告 - 2026-02-25 01:00

## 1. 交易概况
- 总决策数: X
- 实际执行: Y
- 胜率: Z%
- 总盈亏: +$XXX / -$XXX

## 2. 决策质量分析
| 决策ID | 信心 | 预测方向 | 实际结果 | 匹配度 |
|-------|------|---------|---------|-------|
| ... | ... | ... | ... | ... |

## 3. 异常检测
- ⚠️ 连续亏损: 3笔 (XRP SHORT)
- ⚠️ 高信心亏损: 信心85%但亏损$XX
- ✅ 完美执行: 信心→结果匹配度100%

## 4. 市场条件分析
- BTC 1h变化: +0.5%
- 市场状态: [资金溢出]
- 策略适应性: 良好/一般/差

## 5. 数据文件
- Raw数据: decisions_20260225_01.json
- 日志片段: [附关键日志]
```

### 任务2: 每日深度分析与策略优化 (Daily Deep Analysis)

**触发时间**: 每天 00:10 (10 0 * * *)
**执行时长**: ~10-15分钟
**输出**: 
- `docs/openclaw/daily_analysis_YYYYMMDD.md`
- `docs/openclaw/strategy_optimization_YYYYMMDD.md`
- `.msg/prompt_optimized_v{X.X}.yaml` (优化后的策略配置)

#### Phase 1: 数据聚合 (00:10-00:12)

1. **读取过去24小时的所有小时报告**
   ```python
   hourly_reports = glob("docs/openclaw/hourly_analysis_20260225_*.md")
   ```

2. **聚合交易数据**
   ```sql
   -- 24小时统计
   SELECT 
     COUNT(*) as total_trades,
     SUM(CASE WHEN realized_pnl > 0 THEN 1 ELSE 0 END) as wins,
     SUM(CASE WHEN realized_pnl < 0 THEN 1 ELSE 0 END) as losses,
     SUM(realized_pnl) as total_pnl,
     AVG(realized_pnl) as avg_pnl,
     MAX(realized_pnl) as max_win,
     MIN(realized_pnl) as max_loss
   FROM trader_positions
   WHERE created_at >= datetime('now', '-24 hours')
     AND status = 'CLOSED';
   ```

3. **获取市场行情数据**
   ```python
   # 过去24小时的BTC/ETH走势
   # 主要交易对的波动率
   # 资金费率变化
   # 持仓量变化
   ```

#### Phase 2: 深度分析 (00:12-00:14)

##### 分析模块1: 策略表现回顾

| 指标 | 计算方法 | 评估标准 |
|-----|---------|---------|
| **总收益率** | 总盈亏/初始资金 | >5%优秀, 0-5%一般, <0%差 |
| **胜率** | 盈利笔数/总笔数 | >60%优秀, 50-60%一般, <50%差 |
| **盈亏比** | 平均盈利/平均亏损 | >2优秀, 1.5-2一般, <1.5差 |
| **夏普比率** | (收益率-无风险利率)/波动率 | >2优秀 |
| **最大回撤** | 最大连续亏损 | <10%可接受 |
| **Calmar比率** | 年化收益/最大回撤 | >3优秀 |

##### 分析模块2: 策略-市场匹配度

```
市场条件 vs 策略表现矩阵:

               | 狂暴牛市 | 资金溢出 | 阴跌绞杀 | 恶性暴跌
---------------|---------|---------|---------|----------
策略盈利       |   ✅    |   ✅    |   ⚠️    |    ❌
策略亏损       |   ❌    |   ⚠️    |   ❌    |    ⚠️
未触发交易     |   ⚠️    |   ❌    |   ✅    |    ✅

分析: 策略在【资金溢出】状态下表现最佳，在【恶性暴跌】时止损不及时
```

##### 分析模块3: AI决策质量评估

**信心校准分析**:
- 信心80-100%: 应胜率>70%
- 信心60-80%: 应胜率>55%
- 信心<60%: 应谨慎，胜率>40%

**如果实际胜率 < 预期胜率 - 10%**: 
→ Prompt中"信心评估"部分需要优化

##### 分析模块4: 技术指标有效性验证

```python
# 统计各指标触发的交易表现
indicator_performance = {
    "EMA金叉": {"trades": 10, "win_rate": 60%, "avg_pnl": +$5},
    "MACD背离": {"trades": 5, "win_rate": 40%, "avg_pnl": -$3},
    "RSI超买": {"trades": 3, "win_rate": 33%, "avg_pnl": -$8},
}

# 识别失效指标
if indicator_performance["RSI超买"]["win_rate"] < 40%:
    flag_for_removal_or_adjustment("RSI超买")
```

##### 分析模块5: 币种表现分析

```
各币种24小时表现:
- SOL: +5.2% (策略盈利+$12) ✅ 强相关
- XRP: -2.1% (策略亏损-$8)  ❌ 逆势交易
- ETH: +1.5% (策略盈利+$5)  ✅ 正常表现

结论: 策略对SOL的识别能力强，对XRP的判断需改进
```

#### Phase 3: 策略优化建议生成 (00:14-00:15)

##### 优化类型1: Prompt参数微调

**如果发现**:
- 高信心交易频繁亏损 → 提高"最小信心阈值"
- 特定市场状态下表现差 → 增强该状态的"气候判定"逻辑
- 某币种连续亏损 → 加入该币种的"特殊处理规则"

**示例修改**:
```yaml
# 原配置
min_confidence: 76

# 优化后 (基于过去3天高信心交易胜率<60%)
min_confidence: 82  # 提高6个百分点
```

##### 优化类型2: 风控参数调整

**如果发现**:
- 最大回撤超过10% → 降低杠杆或收紧止损
- 持仓时间过长导致亏损 → 加入"时间止损"
- 单笔亏损过大 → 降低仓位比例

**示例修改**:
```yaml
# 原配置
max_positions: 2
btc_eth_max_leverage: 5

# 优化后 (基于回撤分析)
max_positions: 3  # 分散风险
btc_eth_max_leverage: 4  # 降低杠杆
altcoin_max_leverage: 2  # 山寨币更保守
```

##### 优化类型3: 指标权重调整

**如果发现**:
- EMA指标表现优秀，ADX表现差 → 提高EMA权重
- 某个指标频繁产生假信号 → 降低敏感度或移除

**示例修改**:
```yaml
# 在Prompt中加入
indicator_weights:
  EMA: 1.5  # 加权
  MACD: 1.0  # 标准
  ADX: 0.7  # 降低权重
```

##### 优化类型4: 市场状态判定优化

**如果发现**:
- 【资金溢出】状态下频繁亏损 → 重新定义判定条件
- 某状态从未触发 → 简化或移除

**示例修改**:
```yaml
# 原判定
资金溢出: BTC死寂 且 Symbol_ADX≥25 且 1h涨>0.5%

# 优化后 (更严格)
资金溢出: BTC死寂 且 Symbol_ADX≥30 且 1h涨>1.0% 且 Vol>2x均值
```

## ⚠️ 风险控制

1. **自动优化限制**:
   - 不修改核心交易逻辑
   - 不更改API密钥/交易所配置
   - 风控参数调整幅度<30%

2. **人工确认机制**:
   - 所有优化建议需人工确认
   - 高风险变更(如移除指标)需双重确认
   - 提供"一键回滚"功能

3. **监控告警**:
   - 连续3笔亏损立即告警
   - 单日亏损>5%立即告警
   - 策略异常(如无交易)立即告警
